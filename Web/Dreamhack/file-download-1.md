#### 소스코드

**upload** 페이지는 `..`를 필터링한다.
`filename`에는 `..`를 넣을 수 없다.
![[{301B8A97-6355-41BF-AD85-F9A15D07AE7F}.png]]

**read** 페이지는 필터링이 없다.
다운로드되는 파일에 대해 아무런 검사도 하지 않으므로 파일 다운로드 공격에 취약하다.
![[{B0268DA4-CC09-415D-BFC0-7EE27FB802C4}.png]]



#### 해결

read 페이지에서 파일명을 `../flag.py`로 바꿔주면 해결!
